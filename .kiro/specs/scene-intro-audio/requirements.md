# Requirements Document

## Introduction

Before the interleaved VOICE/SFX scene begins in an audio postcard, a short TTS intro clip plays using the George voice. The intro says "Welcome to [city name]. You are now in [eventName]. [atmosphere sentence]" — acting as a calm guide that orients the listener before the immersive scene audio starts. The intro is generated in parallel with the scene lines and ambient soundscape, then prepended to the playlist so it plays first.

## Glossary

- **Intro_Synthesiser**: The function `synthesiseIntro` in `elevenLabsService.ts` responsible for generating the intro TTS clip.
- **George_Voice**: ElevenLabs voice with ID `JBFqnCBsd6RMkjVDRZzb`, used exclusively for the intro clip.
- **Intro_Clip**: The short TTS audio segment that plays before the scene lines begin.
- **Playlist**: The ordered array of object URLs in `playlistRef.current` that the audio postcard plays sequentially.
- **Audio_Postcard_Hook**: The `useAudioPostcard` hook in `useAudioPostcard.ts` that orchestrates scene generation and audio playback.
- **Scene_Lines**: The interleaved VOICE/SFX segments generated by `synthesiseSceneLines`.
- **Ambient_Soundscape**: The looping background audio generated by `generateAmbientSoundscape`.
- **Object_URL**: A `blob:` URL created via `URL.createObjectURL` for in-memory audio playback.

## Requirements

### Requirement 1: Intro Text Synthesis

**User Story:** As a listener, I want to hear a welcoming intro before the scene begins, so that I am oriented to the city and historical event before the immersive audio starts.

#### Acceptance Criteria

1. THE `Intro_Synthesiser` SHALL accept `city` (City object), `eventName` (string), `atmosphere` (string), and an optional `AbortSignal` as parameters.
2. WHEN called, THE `Intro_Synthesiser` SHALL build the intro text as: `"Welcome to ${city.name}. You are now in ${eventName}. ${atmosphere}"`.
3. WHEN the TTS request succeeds, THE `Intro_Synthesiser` SHALL return an Object_URL pointing to the generated audio blob.
4. THE `Intro_Synthesiser` SHALL call the ElevenLabs TTS API using the George_Voice (ID: `JBFqnCBsd6RMkjVDRZzb`) with model `eleven_multilingual_v2`, stability `0.5`, similarity_boost `0.8`, style `0.3`, and use_speaker_boost `true`.
5. IF the TTS request fails for any reason, THEN THE `Intro_Synthesiser` SHALL return `null` without throwing (non-fatal failure).
6. WHEN the `AbortSignal` is aborted before or during the request, THE `Intro_Synthesiser` SHALL return `null` immediately.

### Requirement 2: Parallel Generation

**User Story:** As a listener, I want the intro to be generated without adding extra wait time, so that the audio postcard loads as fast as possible.

#### Acceptance Criteria

1. WHEN the Audio_Postcard_Hook begins audio generation, THE Audio_Postcard_Hook SHALL generate the Intro_Clip, Scene_Lines, and Ambient_Soundscape concurrently using `Promise.all`.
2. WHEN all three generation tasks complete, THE Audio_Postcard_Hook SHALL proceed to playlist assembly regardless of whether the Intro_Clip succeeded or failed.

### Requirement 3: Intro Prepended to Playlist

**User Story:** As a listener, I want the intro to play first before the scene voices, so that I hear the orientation before the immersive scene begins.

#### Acceptance Criteria

1. WHEN the Intro_Clip Object_URL is non-null, THE Audio_Postcard_Hook SHALL prepend it as the first entry in the Playlist.
2. WHEN the Intro_Clip Object_URL is null, THE Audio_Postcard_Hook SHALL assemble the Playlist from Scene_Lines only, with no gap or error.
3. THE Audio_Postcard_Hook SHALL filter out any null entries from the Playlist before playback begins.
4. WHEN playback starts, THE Audio_Postcard_Hook SHALL play the Intro_Clip first, followed by Scene_Lines in their original order, via the existing sequential playlist mechanism.

### Requirement 4: Non-Fatal Intro Failure

**User Story:** As a listener, I want the audio postcard to still work even if the intro fails to generate, so that a TTS error does not block the scene from playing.

#### Acceptance Criteria

1. IF the Intro_Synthesiser returns null, THEN THE Audio_Postcard_Hook SHALL continue to assemble and play the Playlist using Scene_Lines and Ambient_Soundscape only.
2. IF the Intro_Synthesiser returns null, THEN THE Audio_Postcard_Hook SHALL NOT display an error to the user.
3. THE Audio_Postcard_Hook SHALL only surface an error to the user when the Ambient_Soundscape fails to generate.
