# Requirements Document

## Introduction

This feature introduces a **Dual-Mode Audio Experience** to City Whispers. After a user selects a city and era, a Mode Selection screen is shown before entering the audio experience. The user can choose between two modes:

1. **Wander the City** — the existing immersive multi-voice scene with vendors, crowd reactions, and SFX between voices.
2. **Documentary** — a new narrated documentary-style audio experience about what was happening in that city/country/era, generated by Groq and narrated by a single user-chosen voice via ElevenLabs TTS.

Both modes are accessible from within the CityPanel via a mode toggle, allowing the user to switch at any time.

## Glossary

- **App**: The City Whispers single-page application.
- **ModeCard**: The UI component presenting the two mode options as large visual cards.
- **WanderMode**: The existing immersive multi-voice scene audio experience.
- **DocumentaryMode**: The new narrated documentary audio experience.
- **DocumentaryScript**: A 4-segment plain-text script generated by Groq describing world context, city life, key events, and daily culture for the selected city and era.
- **Segment**: One of the 4 parts of a DocumentaryScript, each ≤ 180 words, separated by `---`.
- **VoiceSelector**: A UI component showing 6 narrator voice options for DocumentaryMode.
- **NarratorVoice**: One of the 6 fixed ElevenLabs voices available for DocumentaryMode narration.
- **Playlist**: An ordered sequence of audio URLs played sequentially without gaps between segments.
- **CityPanel**: The main audio playback panel shown during the experience step.
- **ModeToggle**: A tab/toggle control inside CityPanel allowing the user to switch between WanderMode and DocumentaryMode.
- **GroqService**: The service responsible for calling the Groq API.
- **DocumentaryService**: The new service responsible for generating DocumentaryScripts and synthesising documentary audio.
- **ElevenLabsService**: The existing service responsible for ElevenLabs TTS and SFX API calls.

## Requirements

### Requirement 1: Mode Selection Step

**User Story:** As a user, I want to choose between "Wander the City" and "Documentary" modes after selecting a city and era, so that I can decide what kind of audio experience I want.

#### Acceptance Criteria

1. WHEN a user selects an era, THE App SHALL display a ModeCard screen before entering the CityPanel.
2. THE ModeCard SHALL display two large side-by-side cards: one for WanderMode and one for DocumentaryMode.
3. THE ModeCard SHALL display an icon, title, and description for each mode card.
4. WHEN a user selects WanderMode, THE App SHALL navigate directly to the CityPanel in WanderMode.
5. WHEN a user selects DocumentaryMode, THE App SHALL display the VoiceSelector before navigating to the CityPanel.
6. THE ModeCard SHALL display a back button that returns the user to the era selection screen.

---

### Requirement 2: Wander the City Mode (Existing Behaviour)

**User Story:** As a user, I want the existing immersive scene experience to remain available as "Wander the City", so that I can still hear the city as it was with multi-voice scenes and SFX.

#### Acceptance Criteria

1. WHEN WanderMode is active, THE CityPanel SHALL use the existing useAudioPostcard hook to generate and play the multi-voice scene audio.
2. WHILE WanderMode is active, THE CityPanel SHALL display the event name, atmosphere text, and playback controls as before.
3. THE WanderMode card SHALL display a footsteps or compass icon, the title "Wander the City", and the description "Step into the streets. Hear the city as it was."

---

### Requirement 3: Documentary Mode — Script Generation

**User Story:** As a user, I want to hear a narrated documentary about the city and era I selected, so that I can learn about the historical context in an engaging audio format.

#### Acceptance Criteria

1. WHEN DocumentaryMode is selected, THE DocumentaryService SHALL call the Groq API to generate a DocumentaryScript for the selected city, country, and era year.
2. THE DocumentaryScript SHALL consist of exactly 4 segments separated by `---`: world context, city life, key events, and daily life/culture.
3. EACH Segment SHALL be written in documentary narration style and SHALL NOT exceed 180 words.
4. IF the Groq API returns a malformed or empty response, THEN THE DocumentaryService SHALL throw a descriptive error.
5. THE DocumentaryMode card SHALL display the title "Documentary", the description "What was happening in [city] in [era]", and a film or microphone icon.

---

### Requirement 4: Documentary Mode — Voice Selection

**User Story:** As a user, I want to pick a narrator voice for the documentary, so that I can personalise the listening experience.

#### Acceptance Criteria

1. WHEN DocumentaryMode is chosen, THE VoiceSelector SHALL display exactly 6 NarratorVoice options.
2. THE VoiceSelector SHALL display each NarratorVoice with its name and a short description.
3. THE available NarratorVoices SHALL be: George (Warm storyteller), Daniel (Formal broadcaster), Brian (Resonant narrator), Matilda (Professional), Alice (Clear educator), River (Calm, neutral).
4. WHEN a user selects a NarratorVoice and confirms, THE App SHALL navigate to the CityPanel in DocumentaryMode using the chosen voice.
5. THE VoiceSelector SHALL have a default selected voice so the user can proceed without explicitly choosing.

---

### Requirement 5: Documentary Mode — Audio Synthesis

**User Story:** As a user, I want each documentary segment to be narrated by my chosen voice and played as a seamless playlist, so that I hear a continuous documentary experience.

#### Acceptance Criteria

1. WHEN DocumentaryMode audio generation begins, THE DocumentaryService SHALL synthesise each of the 4 Segments as a separate TTS call using the chosen NarratorVoice.
2. THE DocumentaryService SHALL execute all 4 TTS calls in parallel to minimise loading time.
3. THE Playlist SHALL play the 4 synthesised segments sequentially without ambient SFX underneath.
4. IF a TTS call for a Segment fails, THEN THE DocumentaryService SHALL surface a descriptive error to the user.
5. THE CityPanel in DocumentaryMode SHALL display "What was happening in [city] in [era]" as the heading.

---

### Requirement 6: Mode Toggle in CityPanel

**User Story:** As a user, I want to switch between Wander and Documentary modes from within the CityPanel, so that I can change my experience without going back to the start.

#### Acceptance Criteria

1. THE CityPanel SHALL display a ModeToggle (tab or toggle control) that shows both WanderMode and DocumentaryMode options.
2. WHEN a user activates the ModeToggle to switch modes, THE CityPanel SHALL stop the current audio and regenerate audio for the newly selected mode.
3. WHEN switching to DocumentaryMode via the ModeToggle, THE CityPanel SHALL use the previously selected NarratorVoice, or the default voice if none was selected.
4. WHEN switching to WanderMode via the ModeToggle, THE CityPanel SHALL regenerate the immersive scene audio.

---

### Requirement 7: Voice Data

**User Story:** As a developer, I want narrator voice options defined in a central data file, so that voice IDs and metadata are easy to maintain.

#### Acceptance Criteria

1. THE App SHALL define all NarratorVoice options in a dedicated `src/data/voices.ts` file.
2. EACH NarratorVoice entry SHALL include: id (ElevenLabs voice ID), name, and description.
3. THE voices.ts file SHALL export the 6 NarratorVoices as a typed constant array.
