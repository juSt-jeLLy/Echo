# Bugfix Requirements Document

## Introduction

Three polish bugs affecting the scene audio and UI display quality. The audio script (Call 2) ignores the "no narrator" instruction and opens with scene-setting prose instead of immediate sound cues or voices. The atmosphere field (Call 1) returns a verbose 2-3 sentence paragraph when a single short placement sentence is needed. The event name badge in CityPanel is too small to be readable at a glance.

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN the sceneAudioScript is generated by Call 2 THEN the system produces narration-style opening sentences like "The dusty roads are bustling with townspeople discussing..." before any sound cues or voices appear
1.2 WHEN the atmosphere field is generated by Call 1 THEN the system returns a 2-3 sentence descriptive paragraph instead of a single short placement sentence
1.3 WHEN the event name badge is rendered in CityPanel THEN the system displays it at `text-xs` size with light tracking, making it hard to read

### Expected Behavior (Correct)

2.1 WHEN the sceneAudioScript is generated by Call 2 THEN the system SHALL start immediately with a sound cue in [brackets] or a voice shouting — no scene-setting sentences, no narration, no "The streets are..." or "People are..." sentences
2.2 WHEN the atmosphere field is generated by Call 1 THEN the system SHALL return a single sentence of max 15 words in the format "You are [specific location] during [event name]"
2.3 WHEN the event name badge is rendered in CityPanel THEN the system SHALL display it at `text-sm font-semibold` size so it is clearly readable

### Unchanged Behavior (Regression Prevention)

3.1 WHEN the sceneAudioScript is generated THEN the system SHALL CONTINUE TO include local-language phonetic shouts and [sound cues in brackets] throughout the script
3.2 WHEN the atmosphere field is generated THEN the system SHALL CONTINUE TO reference the specific city and the real historical event
3.3 WHEN the event name badge is rendered THEN the system SHALL CONTINUE TO display the Radio icon, uppercase tracking, and primary color styling alongside the badge text
3.4 WHEN the atmosphere text is rendered in CityPanel THEN the system SHALL CONTINUE TO display it below the event badge in a readable style (updated to `text-sm text-muted-foreground` since it is now a short one-liner)
3.5 WHEN Call 1 and Call 2 run THEN the system SHALL CONTINUE TO execute both in parallel and return all four SceneResult fields
