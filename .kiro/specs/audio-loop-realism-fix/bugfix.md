# Bugfix Requirements Document

## Introduction

Two bugs degrade the audio postcard experience. First, the ambient soundscape loops indefinitely even after scene audio ends, or when no scene audio is available — it should only loop while scene audio is playing, and stop (or play once) otherwise. Second, the generated soundscape is too generic and short, failing to evoke the specific street-level experience of the historical scene.

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN scene audio ends THEN the system pauses ambient correctly via `onended`, BUT if the user pauses and resumes after scene audio has ended, the ambient loops forever because `loop = true` is unconditional

1.2 WHEN `sceneAudioUrl` is null (TTS failed or unavailable) THEN the system plays ambient with `loop = true` and no `onended` handler, causing it to loop indefinitely with no way to stop

1.3 WHEN the soundscape is generated THEN the system requests only 5 seconds of audio, producing a clip too short to feel immersive

1.4 WHEN the soundscape prompt is generated by Groq THEN the system instructs the model to produce a generic 40-60 word ambient brief, resulting in non-specific environmental sounds that do not evoke the exact street-level experience of the historical scene

### Expected Behavior (Correct)

2.1 WHEN `sceneAudioUrl` exists THEN the system SHALL set `ambientRef.current.loop = true` so ambient loops during scene audio playback, and the existing `onended` handler on scene audio SHALL pause ambient and reset state

2.2 WHEN `sceneAudioUrl` is null THEN the system SHALL set `ambientRef.current.loop = false` and SHALL attach an `onended` handler to ambient that calls `setIsPlaying(false)`, so ambient plays once and stops

2.3 WHEN the soundscape is generated THEN the system SHALL request `duration_seconds: 10` from the ElevenLabs sound effects API

2.4 WHEN the soundscape prompt is generated by Groq THEN the system SHALL instruct the model to produce a 60-80 word hyper-specific street-level ambient sound design brief describing: surface underfoot (cobblestone/dirt/stone), crowd density, specific vendor types and their sounds, weather conditions, near vs distant sounds, and era-specific machinery or transport — with no voices or speech

### Unchanged Behavior (Regression Prevention)

3.1 WHEN `sceneAudioUrl` exists and scene audio is playing THEN the system SHALL CONTINUE TO play ambient looped at 0.3 volume underneath the scene audio

3.2 WHEN scene audio ends normally THEN the system SHALL CONTINUE TO pause ambient, reset both audio elements to `currentTime = 0`, and set `isPlaying` to false

3.3 WHEN the user pauses during playback THEN the system SHALL CONTINUE TO pause both scene audio and ambient simultaneously

3.4 WHEN the user resumes after pausing (before scene audio has ended) THEN the system SHALL CONTINUE TO resume both audio layers simultaneously

3.5 WHEN ambient soundscape generation fails THEN the system SHALL CONTINUE TO throw an error and not proceed to playback
