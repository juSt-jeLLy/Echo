# Requirements Document

## Introduction

The Documentary mode currently generates 4 segments of 100–180 words each, resulting in 3–5 minutes of narrated audio. This feature shortens the documentary to approximately 1 minute by reducing the segment count to 2 and capping each segment at 60–80 words. The Groq prompt and any segment-count-dependent logic in `documentaryService.ts` and `useDocumentary.ts` are updated accordingly.

## Glossary

- **App**: The City Whispers single-page application.
- **DocumentaryService**: The service in `src/services/documentaryService.ts` responsible for generating documentary scripts via Groq and synthesising audio via ElevenLabs TTS.
- **DocumentaryScript**: A 2-segment plain-text script generated by Groq, covering world context and city life for the selected city and era.
- **Segment**: One of the 2 parts of a DocumentaryScript, each ≤ 80 words, separated by `---`.
- **Playlist**: The ordered sequence of audio URLs played sequentially by `useDocumentary`.
- **GroqService**: The Groq API client used inside DocumentaryService.
- **NarratorVoice**: One of the fixed ElevenLabs voices available for DocumentaryMode narration.

## Requirements

### Requirement 1: Reduced Segment Count

**User Story:** As a user, I want the documentary to be approximately 1 minute long, so that I get a concise audio snapshot without a lengthy wait.

#### Acceptance Criteria

1. THE DocumentaryService SHALL request exactly 2 segments from the Groq API when generating a DocumentaryScript.
2. WHEN the Groq API returns a response, THE DocumentaryService SHALL parse it into exactly 2 non-empty segments separated by `---`.
3. IF the Groq API returns fewer than 2 valid segments, THEN THE DocumentaryService SHALL throw a descriptive error.
4. THE DocumentaryService SHALL NOT pad the segment array beyond the number of segments actually returned by the Groq API.

---

### Requirement 2: Shorter Segment Word Limit

**User Story:** As a user, I want each documentary segment to be concise, so that the total audio stays close to 1 minute.

#### Acceptance Criteria

1. THE DocumentaryService SHALL instruct the Groq API that each Segment must not exceed 80 words.
2. THE Groq prompt SHALL specify 2 segments: Segment 1 covering world context and Segment 2 covering city life.
3. THE Groq prompt SHALL request documentary narration style with no segment labels or headings in the output.

---

### Requirement 3: Playlist Compatibility

**User Story:** As a developer, I want the playback hook to work correctly with 2 segments instead of 4, so that audio plays sequentially without errors.

#### Acceptance Criteria

1. WHEN DocumentaryMode audio generation completes with 2 audio URLs, THE Playlist SHALL play both segments sequentially and stop after the second segment.
2. WHEN the Playlist reaches the end of the 2-segment sequence, THE App SHALL set `isPlaying` to `false`.
3. THE `useDocumentary` hook SHALL function correctly for any playlist length between 1 and 4 segments inclusive.

---

### Requirement 4: Consistent Error Handling

**User Story:** As a user, I want clear feedback if documentary generation fails, so that I can retry without confusion.

#### Acceptance Criteria

1. IF all TTS synthesis calls fail, THEN THE DocumentaryService SHALL throw an error with the message "All documentary audio segments failed to synthesise. Please retry."
2. IF the Groq API returns an empty response, THEN THE DocumentaryService SHALL throw an error with the message "Documentary script generation returned an empty response."
3. IF the Groq API returns fewer than 2 segments, THEN THE DocumentaryService SHALL throw an error indicating the segment count was insufficient.
