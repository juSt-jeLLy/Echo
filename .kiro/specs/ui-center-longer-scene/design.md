# Design Document

## Overview

This feature modifies the layout of two overlay components (`CityPanel` and `EraCard`) to center them on the viewport instead of anchoring them to the bottom, and increases the number of voice lines generated by the Groq service from 5–6 to 15–18 to produce approximately one minute of audio playback.

## Architecture

### Component Changes

**CityPanel.tsx**
- Change outer `div` from `fixed inset-x-0 bottom-0 flex justify-center` to `fixed inset-0 z-20 flex items-center justify-center`
- Change inner card `max-w-2xl` to `max-w-3xl`
- Change inner card padding from `px-6 sm:px-8 py-6` to `px-8 sm:px-10 py-8`
- Keep all other styling, animations, and glassmorphism unchanged

**EraCard.tsx**
- Change outer `div` from `fixed inset-x-0 bottom-0 flex justify-center` to `fixed inset-0 z-20 flex items-center justify-center`
- Inner card already uses `max-w-3xl` — no change needed
- Change inner card padding from `px-6 sm:px-8 py-6` to `px-8 sm:px-10 py-8`
- Keep all other styling, animations, and glassmorphism unchanged

### Service Changes

**groqService.ts**
- In Call 2 (voice line generation), change prompt from "Write 5-6 short voice lines" to "Write 15-18 voice lines"
- Change line length guidance from "5-15 words max" to "5-20 words"
- Change `max_tokens` from 400 to 800 for Call 2
- Keep all other prompt rules unchanged (VOICE:Name|line format, local language, no narration)

## Implementation Details

### Layout Centering

The change from `inset-x-0 bottom-0` to `inset-0` makes the container cover the full viewport. Adding `items-center` to the flex container centers the inner card vertically. The existing `justify-center` already centers horizontally.

The `z-20` ensures the overlay appears above the globe (which has lower z-index). The `pointer-events-none` on the outer container and `pointer-events-auto` on the inner card ensure clicks pass through to the globe outside the card area.

### Audio Scene Length

Increasing from 5–6 lines to 15–18 lines with slightly longer line length (5–20 words instead of 5–15) should produce approximately 60 seconds of audio:
- 15 lines × 12 words average × 0.3 seconds per word ≈ 54 seconds
- 18 lines × 12 words average × 0.3 seconds per word ≈ 65 seconds

The `max_tokens` increase from 400 to 800 ensures the LLM has sufficient token budget to generate all 15–18 lines without truncation.

## Testing Strategy

### Manual Testing

1. **CityPanel centering**: Select a city and era, verify the panel appears centered on screen with globe visible behind it
2. **EraCard centering**: Select a city, verify the era selection card appears centered on screen with globe visible behind it
3. **Audio length**: Generate several audio scenes, verify they run approximately 60 seconds instead of 15–20 seconds
4. **Responsive layout**: Test on mobile and desktop viewports to ensure centering works at all screen sizes

### Build Verification

Run `npm run build` to verify TypeScript compilation and no build errors.

## Rollout Plan

1. Modify `CityPanel.tsx` and `EraCard.tsx` layout
2. Modify `groqService.ts` prompt and token limit
3. Run build verification
4. Manual testing on localhost
5. Deploy to production

## Risks and Mitigations

**Risk**: Centered cards may obscure important globe features
**Mitigation**: The glassmorphism backdrop ensures the globe remains visible. Users can close the card at any time.

**Risk**: Longer audio scenes may increase API costs
**Mitigation**: The increase is 3× the number of lines, but still within reasonable limits for the user experience improvement.

**Risk**: LLM may not consistently generate 15–18 lines
**Mitigation**: The parsing logic already handles variable line counts gracefully. If fewer lines are returned, the scene will be shorter but still functional.
